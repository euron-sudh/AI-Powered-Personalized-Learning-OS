## Features

### Authentication
- Email/password registration and login via Supabase Auth
- **Google OAuth** sign-in on both login and register pages
- **Forgot password** flow — email reset link + set new password page with strength indicator
- Show/hide password toggle on login and reset pages
- JWT-authenticated FastAPI backend with row-level security on all data

### Onboarding
- Multi-step onboarding wizard: grade, curriculum board, subjects, background, marksheet upload
- Board selection: **CBSE, ICSE, Cambridge IGCSE, IB, Common Core**
- Marksheet uploaded to private Supabase Storage bucket

### Dashboard
- Subject cards with progress, chapter counts, and average scores
- **Clickable stat cards** — Subjects navigates to `/learn`, Chapters Done / Average Score scroll to Progress Overview, Total Chapters scrolls to Continue Learning
- Hero section with resume lesson and AI tutor shortcuts
- Progress Overview with per-subject progress bars
- Live updates via **Supabase Realtime**

### AI Curriculum Generation
- Personalized curriculum generated by **Claude Opus 4** per subject and board
- Official syllabus data for **CBSE, ICSE, Cambridge IGCSE, IB, and Common Core**
- Board-specific content style: NCERT format for CBSE, analytical depth for ICSE, command-word style for Cambridge, inquiry-based for IB, CCSS-aligned for Common Core
- 8–12 ordered chapters per subject with titles, descriptions, and learning objectives

### AI Lesson Content
- Rich HTML lesson content generated per chapter (600–900 words)
- **Mermaid.js diagrams** rendered live in-browser
- **LaTeX formulas** rendered via KaTeX
- Key concepts, real-world examples, and chapter summary

### AI Teaching Chat
- Streaming conversational tutor powered by **Claude Opus 4** (Server-Sent Events)
- Socratic teaching method — guides students to answers rather than giving them
- Full conversation history persisted per chapter
- Adaptive explanations based on student responses
- Sentiment-triggered tutor nudges (e.g. "Take a break" when drowsy)

### Activities & Evaluation
- AI-generated activities per chapter: multiple choice, short answer, problem sets
- Board-aligned question style (NCERT format, Cambridge command words, etc.)
- **Claude Opus 4** evaluates responses with: score (0–100), correctness breakdown, detailed feedback, and guidance
- Strength/weakness tracking updates the student learning profile automatically
- Next chapter unlocked on completion

### Adaptive Curriculum Adjustment
- After a score below 60%, **Claude Opus 4** automatically re-orders remaining chapters
- Chapters addressing identified weak topics are promoted earlier in the sequence
- Prerequisites are preserved; re-ordering is transparent to the student

### Voice Tutoring (Speech-to-Speech)
- Real-time bidirectional audio via **OpenAI Realtime API** (WebSocket)
- Server-side Voice Activity Detection (VAD) with 800 ms silence threshold
- Natural interruption — student speech cancels in-progress AI response
- Full conversation transcript (student + AI) with download button
- Lesson-context-aware: AI tutor is briefed on the current chapter title
- Available in-lesson (Voice tab) and as a standalone page (`/voice`)

### Video Sentiment Analysis
- Live webcam feed via WebRTC (no video stored)
- Frames captured every 5 seconds, analyzed by **Claude Vision**
- Detects 6 emotions: **engaged, happy, confused, bored, frustrated, drowsy**
- Results streamed back in real time via **WebSocket** (`/api/video/sentiment/ws`)
- Adaptive actions surfaced in the lesson chat
- All raw frames discarded immediately after classification

### Courses
- Dedicated **Courses page** (`/courses`) with curated external course recommendations
- Grouped by the student's enrolled subjects with subject filter tabs
- Platforms: Khan Academy, Coursera, edX (MIT/Harvard/Stanford), YouTube (3Blue1Brown, Crash Course, etc.)
- Covers Mathematics, Physics, Chemistry, Biology, English, Computer Science, History, Geography, Economics

### Analytics Dashboard
- **Bar chart**: average score per subject (color-coded green/yellow/red)
- **Stacked area chart**: emotion timeline across sessions (30-minute buckets)
- **Donut chart**: emotion distribution across all readings
- Chapter progress bars per subject
- Learning profile: AI-identified strengths and areas for improvement
- **Live updates via Supabase Realtime**

### Profile & Navigation
- Profile dropdown in the nav: avatar, email, Profile Details, My Progress, My Subjects, Edit Preferences, Sign Out
- Dedicated `/profile` page showing account details and learning profile
- Nav tabs: Dashboard · Subjects · Courses · Videos · AI Tutor · Analytics

## Getting Started

### Prerequisites

- Node.js 20+
- Python 3.11+
- Docker Desktop (for Redis)
- A [Supabase](https://supabase.com) project
- An [Anthropic](https://console.anthropic.com) API key
- An [OpenAI](https://platform.openai.com) API key (Realtime API access required)

### 1. Clone and configure

```bash
git clone <repo-url>
cd project4
```

Copy and fill in the environment files (see [Environment Variables](#environment-variables)):

```bash
cp backend/.env.example backend/.env
cp frontend/.env.local.example frontend/.env.local
```

### 2. Start Redis

```bash
docker-compose up -d
```

### 3. Backend setup

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head            # run migrations against Supabase PostgreSQL
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 4. Frontend setup

```bash
cd frontend
npm install
npm run dev                     # http://localhost:3000
```

### 5. Supabase setup

1. Create a project at [supabase.com](https://supabase.com)
2. Copy **Project URL**, **anon key**, **service role key**, and **JWT secret** into your `.env` files
3. Create storage buckets: `marksheets` and `content`
4. Enable Realtime for the `sentiment_logs` table (Database → Replication → Tables)
5. RLS policies are applied automatically via Alembic migrations

### 6. Google OAuth setup (optional)

1. Go to [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials → Create OAuth 2.0 Client ID
2. Add authorized redirect URI: `https://<project-ref>.supabase.co/auth/v1/callback`
3. In Supabase Dashboard → Authentication → Providers → Google → paste Client ID and Secret
4. In Supabase → Authentication → URL Configuration → Redirect URLs, add:
   - `http://localhost:3000/auth/callback`
   - `http://localhost:3000/auth/reset-password`

## Database Schema

All tables managed via Alembic migrations. Supabase Auth handles `auth.users`.

| Table | Purpose |
|---|---|
| `students` | Profile: grade, board, interests, background, marksheet path |
| `subjects` | Per-student subjects with status and difficulty level |
| `chapters` | Ordered chapters with AI-generated `content_json` |
| `activities` | AI-generated quiz/problem sets per chapter |
| `activity_submissions` | Student responses with `evaluation_json` and score |
| `chat_messages` | Full lesson chat history (student + tutor roles) |
| `sentiment_logs` | Emotion, confidence, action_taken per frame analysis |
| `student_progress` | Aggregated progress: chapters done, scores, strengths, weaknesses |
| `student_notes` | Per-chapter freeform notes (auto-saved) |

## AI Integration

### Claude Opus 4 (Anthropic)

| Use case | Approach |
|---|---|
| Curriculum generation | Board-specific guidelines + official syllabus data |
| Chapter content | Rich HTML + Mermaid diagrams + LaTeX formulas, 600–900 words |
| Activity generation | Board-aligned question styles, 4–6 mixed questions per chapter |
| Teaching chat | Socratic method, streaming SSE, full chapter context |
| Activity evaluation | Score + correctness + detailed feedback + strengths/weaknesses |
| Adaptive adjustment | Re-orders remaining chapters by weak topics, preserves prerequisites |
| Sentiment analysis | Claude Vision classifies JPEG frames into 6 emotions with confidence |

### OpenAI Realtime API

- Model: `gpt-4o-realtime-preview`
- Modalities: audio + text
- Voice: `alloy`
- Turn detection: `server_vad`, threshold 0.5, 800 ms silence duration
- Transcription: `whisper-1` for student speech
- FastAPI acts as a transparent bidirectional WebSocket proxy

Any Python ASGI host (Railway, Render, Fly.io, etc.):

```bash
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

Deploy to **Vercel** by connecting the repo, setting the root directory to `frontend/`, and adding the `NEXT_PUBLIC_*` environment variables.

### Redis

Use a managed Redis provider (Redis Cloud, Upstash) in production and update `REDIS_URL`.

## Privacy & Security

- **No video stored**: webcam frames are analyzed and immediately discarded; only emotion labels are persisted
- **Row-Level Security**: all Supabase tables are scoped to the authenticated student
- **JWT verification**: every FastAPI route validates the Supabase JWT before processing
- **Rate limiting**: sentiment analysis (60/min) and curriculum generation (10/min) via slowapi
- **COPPA considerations**: no student PII sent to AI models; marksheets in private Supabase Storage bucket with signed URLs
- **OAuth**: Google sign-in handled entirely through Supabase Auth — no OAuth tokens stored in the app
