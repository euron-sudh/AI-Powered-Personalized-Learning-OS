# LearnOS тАФ AI-Powered Personalized Learning Platform

A warm, gamified, K-12 education platform that adapts in real time using native-audio speech-to-speech conversation (Gemini Live), live video sentiment analysis, and AI-generated personalized curricula. Includes proactive visual tool calls (Mermaid diagrams, Wikipedia images, YouTube clips) mid-conversation, spaced-repetition flashcards, multi-day AI projects, story mode, doubt scanner, podcast-style audio lessons, career glimpses, wellness check-ins with Pomodoro, and a next-best-action coach.

## Features

### Authentication
- Email/password registration and login via Supabase Auth
- **Google OAuth** sign-in on both login and register pages
- **Forgot password** flow тАФ email reset link + set new password page with strength indicator
- Show/hide password toggle on login and reset pages
- JWT-authenticated FastAPI backend with row-level security on all data

### Onboarding
- Multi-step onboarding wizard: grade, curriculum board, subjects, background, marksheet upload
- Board selection: **CBSE, ICSE, Cambridge IGCSE, IB, Common Core**
- Marksheet uploaded to private Supabase Storage bucket

### Dashboard
- Subject cards with progress, chapter counts, and average scores
- **Clickable stat cards** тАФ Subjects navigates to `/learn`, Chapters Done / Average Score scroll to Progress Overview, Total Chapters scrolls to Continue Learning
- Hero section with resume lesson and AI tutor shortcuts
- Progress Overview with per-subject progress bars
- Live updates via **Supabase Realtime**

### AI Curriculum Generation
- Personalized curriculum generated by **Claude Opus 4** per subject and board
- Official syllabus data for **CBSE, ICSE, Cambridge IGCSE, IB, and Common Core**
- Board-specific content style: NCERT format for CBSE, analytical depth for ICSE, command-word style for Cambridge, inquiry-based for IB, CCSS-aligned for Common Core
- 8тАУ12 ordered chapters per subject with titles, descriptions, and learning objectives

### AI Lesson Content
- Rich HTML lesson content generated per chapter (600тАУ900 words)
- **Mermaid.js diagrams** rendered live in-browser
- **LaTeX formulas** rendered via KaTeX
- Key concepts, real-world examples, and chapter summary

### AI Teaching Chat
- Streaming conversational tutor powered by **Claude Opus 4** (Server-Sent Events)
- Socratic teaching method тАФ guides students to answers rather than giving them
- Full conversation history persisted per chapter
- Adaptive explanations based on student responses
- Sentiment-triggered tutor nudges (e.g. "Take a break" when drowsy)

### Activities & Evaluation
- AI-generated activities per chapter: multiple choice, short answer, problem sets
- Board-aligned question style (NCERT format, Cambridge command words, etc.)
- **Claude Opus 4** evaluates responses with: score (0тАУ100), correctness breakdown, detailed feedback, and guidance
- Strength/weakness tracking updates the student learning profile automatically
- Next chapter unlocked on completion

### Adaptive Curriculum Adjustment
- After a score below 60%, **Claude Opus 4** automatically re-orders remaining chapters
- Chapters addressing identified weak topics are promoted earlier in the sequence
- Prerequisites are preserved; re-ordering is transparent to the student

### Voice Tutoring (Speech-to-Speech)
- Real-time bidirectional audio via **Google Gemini Live** (`gemini-2.5-flash-native-audio-preview-09-2025`) тАФ native-audio model, no separate TTS step
- FastAPI backend WebSocket proxy at `/api/voice/gemini` keeps the API key server-side and overlaps the upstream dial with JWT verification to minimize connect latency
- VAD tuned for child pacing: `silenceDurationMs: 1000` тАФ snappy replies without cutting off natural pauses
- `activityHandling: NO_INTERRUPTION` keeps AI turns intact even with speaker-to-mic bleed; mic track is physically disabled while the AI is speaking
- Full conversation transcript (student + AI) with a Devanagari-filter so Indian-accented English never renders in Hindi script
- **Proactive visual tool calls mid-speech** (see AI Integration): the tutor automatically renders a diagram / Wikipedia image / YouTube clip while it's still talking тАФ students learn better with a picture on screen
- Auto-connects the instant the lesson page mounts (parallel with chapter fetch) тАФ tutor is usually ready before the content finishes loading
- Available inside every lesson page (`/learn/[subjectId]/[chapterId]`)

### Chapter Completion
- **Mark a chapter complete** from either the chapter-list row or inside the live lesson
- First completion awards **+25 XP** and increments the legacy `StudentProgress.chapters_completed` aggregate
- Endpoint is idempotent: re-marking a completed chapter is a no-op with no additional XP

### Video Sentiment Analysis
- Live webcam feed via WebRTC (no video stored)
- Frames captured every 5 seconds, analyzed by **Claude Vision**
- Detects 6 emotions: **engaged, happy, confused, bored, frustrated, drowsy**
- Results streamed back in real time via **WebSocket** (`/api/video/sentiment/ws`)
- Adaptive actions surfaced in the lesson chat
- All raw frames discarded immediately after classification

### Courses
- Dedicated **Courses page** (`/courses`) with curated external course recommendations
- Grouped by the student's enrolled subjects with subject filter tabs
- Platforms: Khan Academy, Coursera, edX (MIT/Harvard/Stanford), YouTube (3Blue1Brown, Crash Course, etc.)
- Covers Mathematics, Physics, Chemistry, Biology, English, Computer Science, History, Geography, Economics

### Analytics Dashboard
- **Bar chart**: average score per subject (color-coded green/yellow/red)
- **Stacked area chart**: emotion timeline across sessions (30-minute buckets)
- **Donut chart**: emotion distribution across all readings
- Chapter progress bars per subject
- Learning profile: AI-identified strengths and areas for improvement
- **Live updates via Supabase Realtime**

### Gamification (Wave 1)
- **XP, levels and daily streaks** тАФ earn XP from lessons, quizzes, flashcard reviews; levels gated on cumulative XP
- **Streak freezes** (auto-applied up to freezes-remaining when a day is missed)
- **Daily challenges** тАФ 3 rotating quests (e.g. "review 10 cards", "finish a chapter") with XP rewards
- **Leaderboard** (`/leaderboard`) тАФ global + friends rankings
- **Study buddy** (`/buddy`) тАФ AI companion that cheers progress and nudges on inactivity
- Stats tiles on the dashboard: Subjects ┬╖ Level ┬╖ Day Streak ┬╖ Total XP

### Spaced Repetition (Wave 2)
- **SM-2 flashcard deck** per chapter, auto-generated by Claude on completion
- `/review` тАФ mobile-friendly reveal/rate UI (Again / Hard / Good / Easy тЖТ 1тАУ4)
- Keyboard shortcuts: Space to reveal, 1-4 to rate
- Backfill tool: "Build cards for completed chapters" regenerates missing decks
- Due-card counter surfaced on dashboard and in the next-best-action coach

### Adaptive Practice
- `/practice` тАФ AI-picked mixed quiz drawn from weak topics in the student profile
- Difficulty auto-ramps based on rolling average score

### Immersive Learning (Wave 5)
- **Story mode** (`/story/[chapterId]`) тАФ Claude narrates the chapter as a 5-scene story with a moral
- **Audio podcast** (`/podcast/[chapterId]`) тАФ on-demand MP3 via OpenAI TTS (6 voices), streamable with a download button
- **Career glimpse** (`/career/[chapterId]`) тАФ one-paragraph "what people do with this" snapshot
- **Doubt scanner** (`/scan`) тАФ upload a problem photo (тЙд 8 MB), Claude Vision returns step-by-step reasoning + final answer + underlying concept
- **Projectile physics sim** (`/sim/projectile`) тАФ live canvas simulation with angle/speed sliders and analytical predictions (range, max height, flight time)
- Icon shortcuts (Story / Listen / Careers) injected into the lesson chat panel

### Wellness & Focus (Wave 6)
- **Mood check-in** тАФ 6-mood grid (happy / calm / focused / tired / stressed / stuck) + 1тАУ5 energy slider + optional note
- **Coach response** per mood (e.g. "stuck" тЖТ "step away for 5 minutes, try one simpler example first")
- **Pomodoro timer** with SVG progress ring (25 / 15 / 50 min presets)
- Mood + Pomodoro completion feed the next-best-action coach
- Route: `/focus`

### Project Mode (Wave 6)
- **AI-generated multi-day builds** тАФ pick a subject + chapters + theme, Claude proposes a project with milestones, tasks, and a stretch goal
- Milestone checklist persisted to `localStorage` with a progress bar
- Route: `/project`

### Next-Best-Action Coach (Wave 7)
- **"For you, right now"** widget on the dashboard surfaces up to 3 prioritized cards
- Signals: streak-at-risk, due flashcards, recent mood, rolling quiz average (< 60% = focus area, тЙе 80% = suggest project), missing mood check-in
- Top pick gets a gradient "Top pick" badge

### Visual Design (Warm Adventure Theme)
- Cream/beige palette with parchment-scroll hero banners, glossy gradient stat tiles, and scattered emoji doodles (тнР ЁЯУЪ тЬПя╕П ЁЯМЯ)
- Subject palette colored per discipline (Math blue, Science green, English orange, Coding purple, History amber, etc.)
- Kid-friendly illustrations using emoji + gradients (no external image fetches required)

### Profile & Navigation
- Profile dropdown in the nav: avatar, email, Profile Details, My Progress, My Subjects, Edit Preferences, Sign Out
- Dedicated `/profile` page showing account details and learning profile
- Nav tabs: Dashboard ┬╖ Learn ┬╖ AI Tutor ┬╖ Practice ┬╖ Review ┬╖ Buddy ┬╖ Leaderboard ┬╖ Progress

## Getting Started

### Prerequisites

- Node.js 20+
- Python 3.11+
- Docker Desktop (for Redis)
- A [Supabase](https://supabase.com) project
- An [Anthropic](https://console.anthropic.com) API key (curriculum, teaching, evaluation, Vision)
- A [Google AI Studio](https://aistudio.google.com/apikey) key (Gemini Live voice)
- An [OpenAI](https://platform.openai.com) API key (TTS for the audio-podcast feature only тАФ voice no longer uses OpenAI Realtime)
- A [YouTube Data API v3](https://console.cloud.google.com/apis/library/youtube.googleapis.com) key (optional тАФ enables the `show_video` tool; skip and the tutor will fall back to diagrams only)

### 1. Clone and configure

```bash
git clone <repo-url>
cd project4
```

Copy and fill in the environment files (see [Environment Variables](#environment-variables)):

```bash
cp backend/.env.example backend/.env
cp frontend/.env.local.example frontend/.env.local
```

### 2. Start Redis

```bash
docker-compose up -d
```

### 3. Backend setup

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head            # run migrations against Supabase PostgreSQL
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 4. Frontend setup

```bash
cd frontend
npm install
npm run dev                     # http://localhost:3000
```

> The `dev` script already prepends `cross-env NODE_OPTIONS=--max-http-header-size=65536` тАФ this is required because `@supabase/ssr` cookies can exceed Node's default 8 KB header limit and cause `HTTP 431` on the dev server.

### 5. Supabase setup

1. Create a project at [supabase.com](https://supabase.com)
2. Copy **Project URL**, **anon key**, **service role key**, and **JWT secret** into your `.env` files
3. Create storage buckets: `marksheets` and `content`
4. Enable Realtime for the `sentiment_logs` table (Database тЖТ Replication тЖТ Tables)
5. RLS policies are applied automatically via Alembic migrations

### 6. Google OAuth setup (optional)

## AI Integration

### Claude Opus 4 (Anthropic)

| Use case | Approach |
|---|---|
| Curriculum generation | Board-specific guidelines + official syllabus data |
| Chapter content | Rich HTML + Mermaid diagrams + LaTeX formulas, 600тАУ900 words |
| Activity generation | Board-aligned question styles, 4тАУ6 mixed questions per chapter |
| Teaching chat | Socratic method, streaming SSE, full chapter context |
| Activity evaluation | Score + correctness + detailed feedback + strengths/weaknesses |
| Adaptive adjustment | Re-orders remaining chapters by weak topics, preserves prerequisites |
| Sentiment analysis | Claude Vision classifies JPEG frames into 6 emotions with confidence |

### Google Gemini Live (voice)

- Model: `gemini-2.5-flash-native-audio-preview-09-2025` (pinned тАФ it's the only native-audio revision where tool calling + NO_INTERRUPTION both work cleanly)
- Voice: `Aoede` (soft/warm, child-friendly)
- Modality: audio input тЖТ audio output; text transcripts emitted in parallel via `inputAudioTranscription` / `outputAudioTranscription`
- Turn detection: `automaticActivityDetection` with `silenceDurationMs: 1000` тАФ balances snappy replies with kid-style pauses
- `activityHandling: NO_INTERRUPTION` keeps tutor turns intact against speaker-to-mic bleed; client physically disables the mic track while AI audio is playing as a belt-and-suspenders measure
- `temperature: 0.2` for factual stability
- FastAPI (`voice_gemini.py`) is a transparent WS proxy: auth verification and Gemini dial overlap; client тЖТ Gemini and Gemini тЖТ client run as two concurrent asyncio tasks

### Tool calls wired into the tutor

The voice tutor can render visuals on the lesson stage mid-sentence via Gemini function calls. Every tool call is acked with a `toolResponse.functionResponses` frame so Gemini doesn't stall.

| Tool | Purpose |
|---|---|
| `show_diagram(mermaid_code, title, image_query)` | Render a Mermaid diagram on the lesson stage. `image_query` resolves to a real Wikipedia lead image (`/api/rest_v1/page/summary/{q}`) that appears alongside the diagram. Prompt requires тЙе 6 nodes with emoji labels, classDef-themed colors per subject, and two hierarchy levels. |
| `show_video(query, title, start_seconds?, end_seconds?)` | Search YouTube (via `/api/youtube/search`) for a short embeddable clip and render it inline in a locked-down `youtube.com/embed` iframe. No hallucinated IDs тАФ the backend does the lookup. |
| `show_image(url, title, alt)` | Display a specific Wikimedia Commons URL. Use is rare; `show_diagram(image_query=тАж)` is preferred because it never hallucinates URLs. |

## Privacy & Security

- **No video stored**: webcam frames are analyzed and immediately discarded; only emotion labels are persisted
- **Row-Level Security**: all Supabase tables are scoped to the authenticated student
- **JWT verification**: every FastAPI route validates the Supabase JWT before processing
- **Rate limiting**: sentiment analysis (60/min) and curriculum generation (10/min) via slowapi
- **COPPA considerations**: no student PII sent to AI models; marksheets in private Supabase Storage bucket with signed URLs
- **OAuth**: Google sign-in handled entirely through Supabase Auth тАФ no OAuth tokens stored in the app
